# Autonomous Maintenance Manual

VAMS (VLog Autonomous Maintenance System) は、人間が細かく指示を出さずともプロジェクトが高品質な状態を保ち続けられるようにするための仕組みです。エージェントは以下のガイドラインを参考に、システムの継続的な改善に取り組んでください。

## 1. 定期クリーンアップ (`task maintenance`)

エージェントは隙間時間や定期実行において `task maintenance` を実施し、以下の項目を確認・修正する：

- **Linter (Ruff)**: 違反の自動修正と解消。
- **Tests (pytest)**: 全テストがパスすることの確認。
- **Pointers**: `AGENTS.md` や各ルールファイル内のポインタが有効であるかの検証。

## 2. デッドコードと腐敗の排除

- **未使用コード**: 長期間（1ヶ月以上）使われていない、かつ `Superseded` な ADR に関連するコードを安全に削除。
- **コメント・Fatの排除**: 知らぬ間に追加されたコメントや `try-except` 等の「Fat」を発見し、Zero-Fat 原則に基づいて排除。

## 3. 自己進化するルール

エージェントは、以下の状況を検知した場合に自律的にルールを改善する：

- **ミスの繰り返し**: 似たようなリ修正を 2 回以上行った場合、それをリンタールール（ast-grep等）または `.claude/rules/` に追記。
- **構造の複雑化**: ファイルが 300 行を超えた場合、責任を分割してリファクタリングする計画を人間に提示。

## 4. 履歴の保全 (ADR)

ライブラリの変更や大きなリファクタリングなどアーキテクチャ上の決定を行った際は、未来の開発者（人間・AI問わず）が「なぜその選択をしたのか」を理解できるよう、`docs/adr/` に新しい ADR を発行します。
過去の決定を覆す場合は、文脈が途切れないよう古い ADR のステータスを `Superseded` に変更し、新しい決定へのポインタを記載します。
